# Accessibility audits

Dated manual audit reports live here, named `YYYY-MM-DD-<topic>.md`
(e.g. `2026-06-20-voiceover-editor.md`).

Use the agent prompt library in `docs/agent-build-checklist.md`
(§ Accessibility & inclusion) to produce these. Each report should list findings by
severity (blocker / high / medium), propose label/hint/identifier fixes, and state
confidence rather than claiming WCAG compliance outright.

The rolled-up engineering status is in `../wcag-tracker.md`.
