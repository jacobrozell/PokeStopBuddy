Accessibility Statement

Last updated: 2026-06-16

We aim for WCAG 2.1 AA. PokeStop Buddy is built with accessibility in mind from the first component.

What's implemented

  • VoiceOver labels and identifiers on interactive controls.
  • Dynamic Type via semantic text styles.
  • Minimum 44pt touch targets on primary actions.
  • Status and quality conveyed by icon + text, not color alone.
  • Portrait and landscape on iPhone and iPad.

Known gaps

Manual VoiceOver and large-text device verification are tracked in the project's accessibility audits and completed before each release. This statement is updated as claims are verified on device.
